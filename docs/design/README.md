# Design

本目录维护各模块的长期设计——后端的数据 / 模型设计，以及前端的 UI / 交互设计。

放：

- 后端：模块设计、DBML / 表结构、状态机、跨系统同步方案。
- 前端：设计 token / 主题配置、组件清单与复用约定、页面信息架构、视觉还原对照基线、关键交互流程。

不放：操作步骤和联调手册（那是 [`../runbooks/`](../runbooks/)）、上游原文与设计稿原件（那是 [`../references/`](../references/)）、接口字段 ↔ 展示映射（那是 [`../api/`](../api/)）。

## 维护规则（后端）

- 建议每个模块一个 `<模块>.md`，需要时配 `<模块>.dbml` 描述表结构。
- 改业务实体、状态机、表结构时同步更新这里；涉及关键取舍时在 [`../decisions/`](../decisions/) 新增 ADR。

## 维护规则（前端）

- **设计 token / 主题**：记录主色、间距、圆角、字体等 token 与 `ConfigProvider` 主题配置位置；改 token / 主题时同步更新。
- **组件清单**：记录复用了哪些组件库（如 Ant Design）组件、哪些自研及自研理由——优先复用、避免重复造轮子。
- **视觉还原基线**：保留设计稿 vs 实现的对照截图，作为验证阶段视觉还原的基准（设计稿原件 / 链接放 [`../references/`](../references/)）。
- **关键交互流程**：主路径，以及各交互态（空 / 加载 / 错误 / 无权限）的流转。
- 涉及关键取舍（复用 vs 自研、状态管理、样式方案）时在 [`../decisions/`](../decisions/) 新增 ADR。

设计与代码冲突时，以代码和 [`../overview/implementation-status.md`](../overview/implementation-status.md) 为当前状态来源。
