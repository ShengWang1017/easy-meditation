# Pitfalls

本文件记录已经验证过的坑：症状、原因、修复或规避方式。持续追加，不放尚未验证的猜测。

每条按下面格式写：

## <一句话坑名>

**症状：** 现象、报错、复现条件。

**原因：** 根因，最好定位到具体代码 / 配置 / 环境。

**规避：** 修复或绕过方式；相关操作链接到 [`../runbooks/`](../runbooks/)，相关设计链接到 [`../design/`](../design/)。

> 前端常见坑类目（供参考）：全局样式污染、响应式断点、状态管理时序、SSR / hydration 不一致、组件库（如 Ant Design）版本差异。
