# Overview

本目录是项目总览，新人和 AI 最先读。回答“这个项目做什么、业务长什么样、系统怎么搭、现在做到哪了”。

放：项目目的、业务模型 / 页面地图、架构、技术栈、实现状态。
不放：具体接口字段、临时方案草稿。

## 核心文档

| 文件 | 说明 |
|---|---|
| [`product.md`](./product.md) | 项目目的、用户、核心能力 |
| [`domain-model.md`](./domain-model.md) | 业务实体、关系、状态机、关键规则；前端项目：页面 / 路由地图、核心交互流程与前端状态 |
| [`architecture.md`](./architecture.md) | 系统模块、外部依赖、自建能力、交互边界；前端项目：路由结构、组件分层 |
| [`technical-stack.md`](./technical-stack.md) | 语言、框架、存储、测试、部署等技术栈（前端含 React / 组件库 / 状态管理 / 构建 / 测试） |
| [`implementation-status.md`](./implementation-status.md) | 已实现、部分实现、待办和风险快照 |

[`implementation-status.md`](./implementation-status.md) 是项目“当前状态”的权威来源；文档与代码冲突时，以代码和它为准。
